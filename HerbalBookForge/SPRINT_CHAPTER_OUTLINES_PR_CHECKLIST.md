# HerbalBookForge Chapter Outlines Sprint: PR and Closeout Checklist

## PR Creation
- [ ] Reference all issues created for this sprint in the PR description.
- [ ] Reference all new/updated requirements (HBF.CHO1–HBF.CHO6) in the PR.
- [ ] Summarize major changes: tab renaming, LLM-driven outline generation, editing, regeneration, second-pass consistency-editing, UI structure, tests, and documentation.

## PR Review & Merge
- [ ] Ensure all new/updated requirements are present in both code and requirements documentation.
- [ ] Ensure all new features are fully tested (unit, integration, UI, regression).
- [ ] Document test results in the PR (regression and new features).
- [ ] All issues are linked to the PR and closed on merge.
- [ ] PR is reviewed and approved by the team.

## Sprint Closeout
- [ ] All issues and the PR are closed.
- [ ] User documentation is updated and published.
- [ ] Requirements doc and code comments are in sync.
- [ ] Sprint results are reviewed and lessons learned are captured.
