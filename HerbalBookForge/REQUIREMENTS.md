# HerbalBookForge Requirements

Date: 2026-04-19
Application: HerbalBookForge
Source File: HerbalBookForge.html

## Executive Summary

HerbalBookForge is the herbal-themed baseline authoring application and remains the visual reference for shared shell quality, while preserving domain-specific herbal workflows and governance traceability.


## Functional Requirements

- UIU.HBF.F1: Header navigation shall preserve stable access to all workflow tabs.
- UIU.HBF.F2: Setup workflow shall retain API/model/configuration management plus import/export and backup controls.
- UIU.HBF.F3: Goals, outline, chapter outlines, drafting, prompts, safety, and preview workflows shall remain user-editable.
- UIU.HBF.F4: Agent prompt editing shall remain available with default restoration capability.
- UIU.HBF.F5: Project state persistence and restore behavior shall remain available.

### Chapter Outlines Tab Requirements (Sprint X)

- HBF.CHO1: The Chapter Outlines Tab SHALL display one annotated card or sub-tab per chapter, each representing a chapter outline.
- HBF.CHO2: Each chapter outline SHALL be generated by the Chapter Annotator Agent using the book outline as context.
- HBF.CHO3: User SHALL be able to edit each chapter outline inline and explicitly save changes.
- HBF.CHO4: Chapter Outlines Tab SHALL support per-chapter regeneration via the Chapter Annotator Agent.
- HBF.CHO5: When the last chapter outline is created, the app SHALL resubmit all chapter outlines to the Chapter Annotator Agent for a second-pass consistency-editing step, harmonizing style, structure, and cross-references across chapters.
- HBF.CHO6: The UI structure (cards or sub-tabs) SHALL be clearly documented and implemented consistently.

All requirements above must be present as comments in HerbalBookForge.html and in this document.

## Nonfunctional Requirements

- UIU.HBF.NF1: Layout shall remain responsive across desktop and mobile breakpoints.
- UIU.HBF.NF2: Keyboard focus visibility shall remain present for primary controls.
- UIU.HBF.NF3: Baseline visual identity (emerald palette, typography, tab treatment) shall remain consistent.
- UIU.HBF.NF4: Existing regression selectors and IDs shall remain stable unless intentionally revised with test updates.
